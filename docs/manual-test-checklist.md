# 싸와! 배포본 실사용 테스트 체크리스트

테스트 전 `/status`에서 환경변수 true/false를 확인합니다. 실제 테스트 계정 비밀번호는 코드와 문서에 저장하지 않고 환경변수 또는 별도 보안 채널로 관리합니다.

## 공통

- 비로그인 상태에서 `/app` 접속 시 로그인 화면이 보인다.
- 로그인 성공 후 역할에 맞는 홈으로 이동한다.
- buyer는 admin/supplier 전용 경로에 직접 접근해도 권한 없음 화면이 보인다.
- supplier는 admin 전용 경로에 직접 접근해도 권한 없음 화면이 보인다.
- 데모 초기화 버튼은 admin에게만 보인다.
- 모바일 하단 탭이 역할별로 보인다.

## 구매자

- 홈 상단에 `거래명세서 올리기`, `직접 견적요청`, `지난 구매 다시 견적`이 보인다.
- `거래명세서 올리기`에서 파일 업로드/자동 분석 안내가 보인다.
- `직접 견적요청`에서 단계형 작성 흐름이 열린다.
- 품목 1개만 입력해도 요청 등록이 가능하다.
- 납품 지역, 희망 납품일 누락 시 명확한 오류가 보인다.
- 요청 상세에서 견적 카드의 최저가/빠른납품/추천견적/품목 확인 상태가 보인다.
- 품목별 비교표가 보인다.
- 업체 선택 후 거래가 생성된다.
- 거래 완료 후 구매내역과 구매장부 흐름을 확인할 수 있다.

## 공급업체

- 공급업체 홈에서 오늘 견적 가능한 요청과 제출 대기 건이 보인다.
- 승인 대기 상태에서는 견적 제출이 제한된다.
- 승인된 공급업체는 요청 상세에서 견적을 제출할 수 있다.
- 견적 제출 시 총액, 배송비, 납품 가능일, 세금계산서/카드 가능 여부를 확인한다.
- 제출한 견적이 구매자 요청 상세에 반영된다.
- 선택된 견적과 거래 상태를 공급업체 거래관리에서 확인한다.

## 관리자

- 관리자 홈 상단에 공급업체 승인 대기, 견적 미도착 요청, 사업자 인증 수동검토, 신고/피드백, 환경 상태가 보인다.
- 공급업체 승인/보완/반려 흐름이 동작한다.
- 사업자 인증 실패 또는 수동검토 건을 확인할 수 있다.
- 견적 미도착 요청에서 수동 매칭 후보를 확인할 수 있다.
- 환경 점검 화면에서 `/status` JSON을 열 수 있다.
- 데모 데이터 관리와 초기화는 관리자에게만 보인다.

## 모바일

- 첫 CTA가 화면 상단에서 보인다.
- 하단 탭은 buyer 기준 `홈`, `견적`, `거래`, `구매장부`, `알림` 순서다.
- 견적 비교표는 가로 스크롤로 볼 수 있다.
- 단계형 입력 화면에서 버튼과 입력칸이 겹치지 않는다.
