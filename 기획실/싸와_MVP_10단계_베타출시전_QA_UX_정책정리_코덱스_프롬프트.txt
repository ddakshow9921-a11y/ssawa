싸와! MVP 10단계 코덱스 작업지시서
주제: 베타 출시 전 품질 점검 + 모바일 UX 최종 고도화 + 온보딩/약관/QA 준비

너는 최고의 시니어 풀스택 개발자이자 UX 기획자다.

우리는 B2B 자재 견적구매 플랫폼 “싸와!” 앱을 개발 중이다.

싸와!는 음식점, 카페, 매장, 공사현장, 사업자가 필요한 식자재, 포장재, 소모품, 설비자재, 건축자재 등을 올리면 지역 공급업체들이 견적을 제출하고, 구매자는 여러 견적을 비교해 업체를 선택할 수 있는 앱이다.

이번 10단계는 신규 대형 기능을 무리하게 추가하는 단계가 아니다.

이번 단계의 핵심은 지금까지 만든 1~9단계 기능을 실제 베타 테스트 가능한 수준으로 안정화하는 것이다.

즉, 다음을 정리한다.

- 모바일 반응형 최종 점검
- 온보딩/튜토리얼
- 빈 상태/로딩/에러 화면
- 접근성/사용성 개선
- 랜딩페이지/메타데이터 정리
- 약관/개인정보처리방침/운영정책 기본 화면
- 관리자 QA 체크리스트
- 베타 테스트 준비
- 성능/빌드/라우팅 오류 점검
- 배포 전 최종 검수

3. 전체 메뉴 구조 정리
────────────────────────────────

앱 메뉴 구조가 너무 복잡해지지 않게 정리한다.

구매자 주요 메뉴:

- 홈
- 견적요청
- 거래내역
- 자료분석
- 구매내역/장부대기
- 알림
- 내 정보

공급업체 주요 메뉴:

- 공급업체 홈
- 견적요청 찾기
- 제출한 견적
- 거래관리
- 문의/메시지
- 정산/요금제
- 신뢰도/후기
- 알림
- 업체 정보

관리자 주요 메뉴:

- 관리자 홈
- 견적요청 관리
- 공급업체 관리
- 거래 관리
- 분석 관리
- 알림/메시지 관리
- 신고/분쟁 관리
- 후기/신뢰도 관리
- 수익/정산 관리
- 운영 품질
- 설정

모바일에서는 모든 메뉴를 한 번에 보여주지 말고, 핵심 메뉴 중심으로 구성한다.

구매자 모바일 하단 탭 추천:

- 홈
- 견적
- 거래
- 자료
- 알림

공급업체 모바일 하단 탭 추천:

- 홈
- 요청
- 견적
- 거래
- 알림

관리자는 PC 중심이지만 모바일에서도 깨지지 않게 한다.

────────────────────────────────
4. 온보딩/튜토리얼
────────────────────────────────

처음 접속한 사용자가 싸와!를 이해할 수 있도록 온보딩 화면을 만든다.

권장 경로:

- /app/onboarding
- /app/buyer/onboarding
- /app/supplier/onboarding

구매자 온보딩 4단계:

1. 필요한 자재를 올리세요
   - 사진, 거래명세서, 문장으로 쉽게 견적요청을 만들 수 있습니다.

2. 업체들이 견적합니다
   - 지역과 카테고리에 맞는 공급업체가 견적을 제출합니다.

3. 비교하고 선택하세요
   - 금액, 배송일, 세금계산서, 후기, 신뢰도를 비교하세요.

4. 구매내역은 장부로 정리됩니다
   - 거래 완료 후 구매내역을 오늘장사 장부에 반영할 수 있습니다.

공급업체 온보딩 4단계:

1. 입점 신청하세요
   - 사업자 정보와 취급 카테고리를 등록합니다.

2. 지역 견적요청을 받아보세요
   - 납품 가능 지역과 카테고리에 맞는 요청을 확인합니다.

3. 빠르게 견적하세요
   - 빠른 응답과 정확한 견적은 선택 확률을 높입니다.

4. 거래와 정산을 관리하세요
   - 거래 상태, 후기, 신뢰도, 수수료 내역을 확인합니다.

온보딩은 최초 1회만 표시할 수 있도록 localStorage 또는 user profile flag를 활용한다.

필드 예시:

- onboarding_completed
- onboarding_completed_at

MVP에서는 localStorage 기반이어도 된다.

────────────────────────────────
5. 빈 상태 화면 정리
────────────────────────────────

각 화면에 데이터가 없을 때 빈 화면이 어색하지 않아야 한다.

공통 EmptyState 컴포넌트를 만든다.

필수 props:

- title
- description
- actionLabel
- actionHref 또는 onAction
- icon 또는 visual
- variant

적용할 화면:

구매자:
- 견적요청 없음
- 도착한 견적 없음
- 거래내역 없음
- 구매내역 없음
- 분석 이력 없음
- 알림 없음
- 문의 없음
- 신고 내역 없음

공급업체:
- 매칭 요청 없음
- 제출한 견적 없음
- 거래 없음
- 정산 내역 없음
- 후기 없음
- 알림 없음

관리자:
- 승인 대기 업체 없음
- 신고 없음
- 분쟁 없음
- 분석 실패 없음
- 정산 대기 없음

예시 문구:

견적요청 없음:
> 아직 견적요청이 없어요.
> 필요한 자재를 올리면 업체들이 견적을 보내드립니다.

도착한 견적 없음:
> 아직 견적이 도착하지 않았어요.
> 매칭된 업체들이 확인 중입니다.

공급업체 요청 없음:
> 현재 조건에 맞는 요청이 없습니다.
> 납품 지역이나 카테고리 설정을 확인해보세요.

알림 없음:
> 새 알림이 없습니다.
> 견적이나 거래 상태가 바뀌면 이곳에서 알려드릴게요.

────────────────────────────────
6. 로딩/에러 상태 정리
────────────────────────────────

공통 LoadingState, ErrorState 컴포넌트를 만든다.

로딩 상태:

- 페이지 로딩
- 리스트 로딩
- 카드 로딩 skeleton
- 버튼 처리 중
- 파일 분석 중
- 견적 제출 중
- 거래 상태 변경 중

에러 상태:

- 데이터를 불러오지 못함
- 저장 실패
- 권한 없음
- 네트워크 오류
- 파일 업로드 실패
- 분석 실패
- 잘못된 접근

ErrorState props:

- title
- description
- retryLabel
- onRetry
- homeHref
- detail

에러 문구 예시:

> 정보를 불러오지 못했습니다.
> 잠시 후 다시 시도해주세요.

저장 실패:
> 저장 중 문제가 발생했습니다.
> 입력한 내용을 확인하고 다시 시도해주세요.

권한 없음:
> 접근 권한이 없습니다.
> 이 화면은 해당 역할의 사용자만 볼 수 있습니다.

────────────────────────────────
7. 권한/승인 상태 화면
────────────────────────────────

공급업체 승인 상태별 화면을 더 명확히 정리한다.

pending:
> 입점 신청 검토 중입니다.
> 승인 후 견적요청에 참여할 수 있습니다.

needs_revision:
> 보완이 필요한 항목이 있습니다.
> 관리자 안내에 따라 정보를 수정해주세요.

rejected:
> 입점 신청이 반려되었습니다.
> 사유를 확인하고 재신청할 수 있습니다.

suspended:
> 현재 이용이 제한된 상태입니다.
> 자세한 내용은 운영팀 안내를 확인해주세요.

buyer/supplier/admin 권한 분기에서 잘못된 화면에 접근하면 공통 권한 없음 화면을 보여준다.

────────────────────────────────
8. 404/잘못된 접근 화면
────────────────────────────────

404 페이지 또는 not-found 화면을 정리한다.

문구:

> 페이지를 찾을 수 없습니다.
> 주소가 잘못되었거나 이동된 페이지입니다.

버튼:

- 홈으로 가기
- 이전 페이지로 돌아가기

싸와! 브랜드 톤에 맞게 너무 딱딱하지 않게 구성한다.

────────────────────────────────
9. 랜딩페이지 최종 정리
────────────────────────────────

싸와! 홈/랜딩 화면 문구를 정리한다.

메인 카피:

> 필요한 자재를 올리면, 업체들이 견적합니다.

서브 카피:

> 식자재, 포장재, 소모품, 설비자재까지 사업자 구매견적을 한 번에 비교하세요.

CTA:

- 견적요청 시작하기
- 공급업체 입점하기

섹션 구성:

1. 메인 히어로
2. 구매자 이용 흐름
   - 올리기 → 견적받기 → 비교하기 → 거래하기 → 장부정리
3. 공급업체 이용 흐름
   - 입점하기 → 요청받기 → 견적하기 → 거래하기
4. 주요 카테고리
   - 식자재
   - 포장재
   - 소모품
   - 주방용품
   - 설비/닥트/환기자재
   - 건축자재
   - 공구/산업자재
5. 오늘장사 연동 가치
   - 구매내역 자동정리
   - 매입비 리포트
   - 세금자료 준비
6. 베타 테스트 안내
7. FAQ
8. 하단 정책 링크

FAQ 예시:

Q. 구매자는 무료인가요?
A. 베타 기간 동안 구매자 견적요청은 무료로 제공합니다.

Q. 공급업체 입점비가 있나요?
A. 초기 베타 기간에는 무료 입점을 지원합니다. 추후 요금제는 별도 안내됩니다.

Q. 실제 결제도 앱에서 하나요?
A. 초기 MVP에서는 견적과 거래 상태 관리를 중심으로 제공하며, 결제 기능은 추후 단계적으로 연동됩니다.

Q. 세금계산서 발행도 가능한가요?
A. 세금계산서 가능 업체를 확인할 수 있으며, 실제 발행은 공급업체와 협의합니다. 추후 자동화 연동을 준비하고 있습니다.

keywords:

18. 상태 라벨/배지 통일
────────────────────────────────

공통 상태 라벨 유틸을 만든다.

예시:

quote_request status:
- open: 견적 받는 중
- quoted: 견적 도착
- selected: 업체 선택
- in_progress: 거래 진행 중
- completed: 완료
- cancelled: 취소

deal status:
- pending_confirmation: 확인 대기
- confirmed: 거래 확정
- preparing: 납품 준비
- delivering: 납품 중
- delivered: 납품 완료
- completed: 거래 완료
- disputed: 문제 발생

analysis status:
- uploaded: 업로드 완료
- analyzing: 분석 중
- needs_review: 검토 필요
- completed: 분석 완료
- failed: 분석 실패

supplier approval:
- pending: 승인 대기
- approved: 승인 완료
- needs_revision: 보완 요청
- rejected: 반려
- suspended: 제한

모든 화면에서 같은 한국어 라벨과 배지 스타일을 사용한다.

19. 빌드/린트/타입 오류 점검

- npm run build

────────────────────────────────
25. 최종 목표 기억
────────────────────────────────

싸와!는 단순 견적앱이 아니다.

싸와!의 핵심은 다음이다.

> 구매자가 필요한 자재를 쉽게 올린다.
> 공급업체가 견적을 제출한다.
> 구매자가 비교 후 업체를 선택한다.
> 거래가 진행되고 구매내역이 정리된다.
